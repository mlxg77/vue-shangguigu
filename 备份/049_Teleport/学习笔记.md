# 049 Teleport

## 本节目标

掌握 `Teleport`——将组件的 DOM 内容"传送"到指定位置，脱离当前组件的 DOM 层级。

## 核心知识点

### 1. 为什么需要 Teleport？

弹窗（Modal）组件通常嵌套在父组件内部：

```
<div class="outer" style="filter: saturate(200%)">
  <Modal />     <!-- 弹窗在 outer 内部 -->
</div>
```

问题：父组件的 CSS（如 `filter`、`overflow: hidden`、`transform`）会影响子组件中的弹窗，导致弹窗样式异常。

### 2. Teleport 的用法

```html
<!-- Modal.vue -->
<template>
  <button @click="isShow = true">展示弹窗</button>

  <teleport to="body">
    <div class="modal" v-show="isShow">
      <h2>我是弹窗的标题</h2>
      <p>我是弹窗的内容</p>
      <button @click="isShow = false">关闭弹窗</button>
    </div>
  </teleport>
</template>
```

- `to="body"`：将 `teleport` 内的内容传送到 `<body>` 标签下
- 弹窗的 DOM 不再在 `outer` 内部，而是直接挂载到 `body`

### 3. 效果对比

**不用 Teleport**：
```
<body>
  <div class="outer" style="filter: saturate(200%)">
    <div class="modal">...</div>  <!-- 受 filter 影响 -->
  </div>
</body>
```

**使用 Teleport**：
```
<body>
  <div class="outer" style="filter: saturate(200%)">
    <!-- modal 不在这里了 -->
  </div>
  <div class="modal">...</div>     <!-- 直接在 body 下，不受 outer 的 CSS 影响 -->
</body>
```

### 4. Teleport 的特点

| 特点 | 说明 |
|------|------|
| `to` 属性 | 指定目标选择器（如 `body`、`#app`、`.some-class`） |
| DOM 位置改变 | 渲染后的 DOM 移到目标位置 |
| 逻辑不变 | 组件的逻辑（数据、事件）仍然属于原组件 |
| CSS 隔离 | 不受父组件 CSS 影响 |

### 5. App.vue 中的使用

```html
<!-- App.vue -->
<div class="outer">
  <h2>我是App组件</h2>
  <img src="..." alt="">
  <Modal/>   <!-- Modal 组件在这里使用 -->
</div>
```

> App 组件的 `.outer` 设置了 `filter: saturate(200%)`，如果不使用 Teleport，弹窗会受影响。
> 使用 `<teleport to="body">` 后，弹窗直接挂在 body 下，不受 `filter` 影响。
