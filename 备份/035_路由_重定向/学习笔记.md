# 035 路由：重定向

## 本节目标

配置路由重定向——访问某路径时自动跳转到另一个路径。

## 相对 034 的变化

路由配置中新增了一条重定向规则：

```ts
{
  path: '/',
  redirect: '/home'  // 访问 / 时自动跳到 /home
}
```

## 核心知识点

### 1. 重定向配置

```ts
routes: [
  { path: '/home', component: Home },
  { path: '/news', component: News },
  { path: '/about', component: About },
  { path: '/', redirect: '/home' }  // ← 重定向
]
```

访问 `http://localhost:5173/` → 自动跳到 `http://localhost:5173/home`

### 2. 为什么需要重定向

没有重定向时，访问根路径 `/` 会匹配不到任何路由，页面空白。重定向让用户进入网站时**默认看到首页**。

### 3. 重定向的几种写法

```ts
// 写法一：字符串
{ path: '/', redirect: '/home' }

// 写法二：命名路由
{ path: '/', redirect: { name: 'zhuye' } }

// 写法三：函数
{ path: '/', redirect: (to) => '/home' }
```
