# 039 修改数据（三种方式）

## 本节目标

掌握 Pinia 中修改 store 数据的三种方式，理解 actions 的用法。

## 相对 038 的变化

- store 中新增了 `actions` 配置
- 组件的方法体从空变成了实际调用
- LoveTalk 的请求逻辑也搬到了 store 的 action 中

## 核心知识点

### 三种修改方式

**方式一：直接修改**
```ts
countStore.sum += 1
```
简单粗暴，适合修改单个属性。

**方式二：$patch 批量修改**
```ts
countStore.$patch({
  sum: 888,
  school: '尚硅谷',
  address: '北京'
})
```
一次修改多个属性，适合批量更新。

**方式三：调用 actions（推荐）**
```ts
countStore.increment(n.value)
```
在 store 中定义 actions，组件调用方法。

### actions 的写法

```ts
// store/count.ts
export const useCountStore = defineStore('count', {
  actions: {
    increment(value: number) {
      console.log('increment被调用了', value)
      if (this.sum < 10) {   // ← this 指向 store 实例
        this.sum += value
      }
    }
  },
  state() {
    return { sum: 6, school: 'atguigu', address: '宏福科技园' }
  }
})
```

**关键点：**
- actions 中的方法通过 `this.sum` 访问 state
- `this` 指向 store 实例（和 Vue 2 的 methods 类似）
- 可以写业务逻辑（如条件判断）

### LoveTalk 的 action（异步请求）

```ts
// store/loveTalk.ts
actions: {
  async getATalk() {
    let { data: { content: title } } = await axios.get('...')
    let obj = { id: nanoid(), title }
    this.talkList.unshift(obj)
  }
}
```

组件中调用：
```ts
function getLoveTalk() {
  talkStore.getATalk()  // 调用 store 的 action
}
```

### 三种方式对比

| 方式 | 写法 | 适用场景 |
|------|------|----------|
| 直接修改 | `store.sum += 1` | 简单的单属性修改 |
| $patch | `store.$patch({...})` | 批量修改多个属性 |
| actions | `store.xxxMethod()` | 有业务逻辑、需要复用（推荐） |

### 为什么推荐 actions

- 业务逻辑集中在 store，不散落在组件里
- 多个组件可以复用同一个 action
- 方便维护和测试

## 与 Vuex 的对比

Vuex 修改数据必须走 `mutation → commit`，同步/异步分两套。Pinia 的 actions **同步异步都能写**，直接调用方法即可。
