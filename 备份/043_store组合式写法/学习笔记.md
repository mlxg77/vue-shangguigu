# 043 store 组合式写法

## 本节目标

掌握 Pinia 的组合式写法——用 setup 函数风格写 store，替代 Options 风格。

## 相对 042 的变化

loveTalk.ts 从 Options 式改为**组合式**：

```ts
// 042（Options 式）
export const useTalkStore = defineStore('talk', {
  state() { return { talkList: [...] } },
  actions: { async getATalk() { this.talkList.unshift(obj) } }
})

// 043（组合式）
export const useTalkStore = defineStore('talk', () => {
  const talkList = reactive([...])       // ← state
  async function getATalk() { ... }      // ← actions
  return { talkList, getATalk }          // ← 暴露
})
```

## 核心知识点

### 1. 组合式写法的结构

```ts
import { defineStore } from 'pinia'
import { reactive } from 'vue'

export const useTalkStore = defineStore('talk', () => {
  // ① 数据（相当于 state）
  const talkList = reactive(
    JSON.parse(localStorage.getItem('talkList') as string) || []
  )

  // ② 方法（相当于 actions）
  async function getATalk() {
    let { data: { content: title } } = await axios.get('...')
    let obj = { id: nanoid(), title }
    talkList.unshift(obj)  // ← 直接用变量，不用 this
  }

  // ③ 返回（暴露给组件使用）
  return { talkList, getATalk }
})
```

### 2. 组合式 vs Options 式

| | Options 式 | 组合式 |
|--|-----------|--------|
| 第二个参数 | `{ state, getters, actions }` | `() => { ... }` 函数 |
| 数据 | `state()` 返回 | `ref()` / `reactive()` |
| 计算属性 | `getters: {}` | `computed()` |
| 方法 | `actions: {}` | 普通函数 |
| 访问数据 | `this.sum` | 直接用变量名 `sum` |
| this | 有 | **没有** |

### 3. 组合式写法的优势

- **不需要 this**——直接用闭包变量，更符合 JS 直觉
- **更好的 TS 支持**——类型推导更自然
- **更灵活**——可以用任何 Composition API（watch、computed、onMounted 等）
- **和组件写法一致**——组件用 `<script setup>`，store 用 setup 函数，风格统一

### 4. getters 在组合式写法中怎么写

```ts
export const useCountStore = defineStore('count', () => {
  let sum = ref(6)
  let bigSum = computed(() => sum.value * 10)  // ← computed 代替 getters

  function increment(value: number) {
    if (sum.value < 10) sum.value += value
  }

  return { sum, bigSum, increment }
})
```

### 5. 组件使用方式完全不变

```ts
// 不管 store 是 Options 式还是组合式，组件里的用法都一样
const talkStore = useTalkStore()
const { talkList } = storeToRefs(talkStore)

function getLoveTalk() {
  talkStore.getATalk()
}
```

## 与 006 的呼应

006 的 `<script setup>` 简化了组件写法；043 的 setup 函数简化了 store 写法——思想完全一致。
