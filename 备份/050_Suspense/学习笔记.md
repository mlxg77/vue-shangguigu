# 050 Suspense

## 本节目标

掌握 `Suspense`——在异步组件加载时显示 fallback 内容，优化用户体验。

## 核心知识点

### 1. 异步组件的问题

Child.vue 中使用了顶层 `await`：

```ts
// Child.vue
let sum = ref(0)
let { data: { content } } = await axios.get('https://api.uomg.com/api/rand.qinghua?format=json')
console.log(content)
```

> 顶层 `await` 使组件变成了**异步组件**。异步组件在数据请求完成前无法渲染，Vue 会报错——需要用 `Suspense` 包裹。

### 2. Suspense 的用法

```html
<!-- App.vue -->
<Suspense>
  <template v-slot:default>
    <Child/>
  </template>
  <template v-slot:fallback>
    <h2>加载中......</h2>
  </template>
</Suspense>
```

| 插槽 | 作用 |
|------|------|
| `default` | 正常渲染的内容（异步组件） |
| `fallback` | 异步组件加载期间显示的"占位"内容 |

### 3. 工作流程

```
1. App 渲染 → Suspense 开始
2. Child 是异步组件 → 显示 fallback（"加载中..."）
3. await axios.get(...) 完成
4. Child 渲染完成 → 替换 fallback，显示 Child 内容
```

### 4. 本节代码说明

```html
<!-- App.vue -->
<Suspense>
  <template v-slot:default>
    <Child/>
  </template>
  <!-- fallback 被注释掉了，所以加载期间什么都不显示 -->
  <!-- <template v-slot:fallback> -->
    <!-- <h2>加载中......</h2> -->
  <!-- </template> -->
</Suspense>
```

```ts
// Child.vue
import { ref } from 'vue'
import axios from 'axios'

let sum = ref(0)
// 顶层 await：组件变为异步组件
let { data: { content } } = await axios.get('https://api.uomg.com/api/rand.qinghua?format=json')
console.log(content)
```

> 如果不使用 `Suspense` 包裹含有顶层 `await` 的组件，Vue 会报错：
> `Component <Child>: setup function returned a promise, but no <Suspense> boundary was found`

### 5. Suspense 的特点

- 用于包裹**异步组件**（含顶层 `await` 的组件）
- 提供 `default` 和 `fallback` 两个插槽
- `fallback` 在异步操作进行时显示
- 异步操作完成后自动切换到 `default`

### 6. 什么情况会产生异步组件？

| 方式 | 示例 |
|------|------|
| 顶层 `await` | `let data = await axios.get(...)` |
| `defineAsyncComponent` | `const Child = defineAsyncComponent(() => import('./Child.vue'))` |
