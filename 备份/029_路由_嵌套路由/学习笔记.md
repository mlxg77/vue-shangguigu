# 029 路由：嵌套路由

## 本节目标

在一个路由组件内部再嵌套子路由——News 页面里嵌套 Detail 详情页。

## 相对 028 的变化

新增了 `pages/Detail.vue`，并在 News 路由下配置了 `children`：

```ts
{
  name: 'xinwen',
  path: '/news',
  component: News,
  children: [              // ← 嵌套路由
    {
      path: 'detail',      // ← 注意：不加 /
      component: Detail
    }
  ]
}
```

## 核心知识点

### 1. children 配置

在父路由规则中加 `children` 数组，里面是子路由规则：

```ts
{
  path: '/news',
  component: News,
  children: [
    { path: 'detail', component: Detail }
  ]
}
```

### 2. 子路由的 path 不加 `/`

```ts
// ✅ 正确：path 不加 /
{ path: 'detail', component: Detail }
// 完整路径自动拼接为 /news/detail

// ❌ 错误：path 加了 /
{ path: '/detail', component: Detail }
// 变成绝对路径 /detail，脱离了父路由
```

### 3. 父组件中需要 `<RouterView>`

News.vue 中要有 RouterView 来展示子路由组件：

```html
<!-- News.vue -->
<div class="news">
  <ul>
    <li v-for="news in newsList" :key="news.id">
      <RouterLink to="/news/detail">{{ news.title }}</RouterLink>
    </li>
  </ul>
  <div class="news-content">
    <RouterView />   <!-- ← Detail 组件在这里渲染 -->
  </div>
</div>
```

### 4. 嵌套路由的 URL

```
/news         → News 组件
/news/detail  → News 组件 + Detail 组件（在 News 的 RouterView 里）
```

访问 `/news/detail` 时，**News 和 Detail 同时渲染**——News 是外壳，Detail 是内容。

### 5. 嵌套层级

```
App.vue 的 <RouterView>
  └─ News.vue 的 <RouterView>
       └─ Detail.vue
```

可以无限嵌套，每一层父组件都需要自己的 `<RouterView>`。
