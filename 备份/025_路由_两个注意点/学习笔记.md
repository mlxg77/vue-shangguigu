# 025 路由：两个注意点

## 本节目标

了解路由组件的存放位置和组件拆分的规范。

## 相对 024 的变化

| 变化点 | 024 | 025 |
|--------|-----|-----|
| 路由组件位置 | `components/` | `pages/`（规范调整） |
| 头部标题 | 直接写在 App.vue | 拆成 `components/Header.vue` |

## 核心知识点

### 注意点一：路由组件放 pages/ 目录

```ts
// router/index.ts
import Home from '../pages/Home.vue'    // ← pages/ 不是 components/
import News from '../pages/News.vue'
import About from '../pages/About.vue'
```

**约定：**
- `pages/`（或 `views/`）：放**路由组件**（通过 URL 切换的页面级组件）
- `components/`：放**非路由组件**（被其他组件引入使用的通用组件，如 Header）

> 这是社区约定，不是强制规则，但大项目都会遵循。

### 注意点二：路由组件通常不写 name

```ts
// pages/Home.vue
<script setup lang="ts" name="Home">  // ← 可以写，但通常不需要
```

路由组件一般不需要 `name` 属性（因为它们通过路由配置而非组件名引用）。非路由组件（如 Header）写 name 方便调试。

### 组件拆分

025 把标题拆成了独立的 `components/Header.vue`：

```html
<!-- App.vue -->
<Header/>           <!-- 非路由组件：固定不变的头部 -->
<div class="navigate">
  <RouterLink ...>首页</RouterLink>
</div>
<RouterView />       <!-- 路由组件：根据 URL 变化的内容区 -->
```

> 把固定不变的部分（Header）和变化的部分（RouterView）分开，是组件设计的基本原则。
