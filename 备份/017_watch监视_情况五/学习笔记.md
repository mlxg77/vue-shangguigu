# 017 watch 监视：情况五

## 本节目标

同时监视**多个数据**——用数组把多个数据源放在一起。

## 相对 016 的变化

016 监视单个属性；017 同时监视多个数据（name + car）。

## 核心知识点

### 1. 用数组监视多个数据

```ts
watch(
  [() => person.name, person.car],  // ① 数组：多个数据源
  (newValue, oldValue) => {          // ② 回调：收到的是数组
    console.log('变化了', newValue, oldValue)
  },
  { deep: true }                      // ③ 配置
)
```

### 2. newValue 和 oldValue 是数组

```ts
watch([() => person.name, person.car], (newValue, oldValue) => {
  // newValue = ['张三~', { c1: '奥迪', c2: '宝马' }]
  // oldValue = ['张三', { c1: '奔驰', c2: '宝马' }]
  
  // 可以解构
  let [newName, newCar] = newValue
})
```

数组中每个元素对应一个数据源的变化值，顺序和第一个参数的数组一致。

### 3. 数组里可以混合不同类型的数据源

```ts
watch(
  [() => person.name,   // getter 函数（情况四的写法）
   person.car],          // 直接传 reactive 子对象
  (newValue, oldValue) => {},
  { deep: true }
)
```

- `() => person.name` — 基本类型属性，用 getter
- `person.car` — 对象类型，直接传（配合 deep）

### 4. 配置对象对数组中所有数据源生效

`{ deep: true }` 会对数组中所有对象类型的数据源开启深度监视。

## watch 五种情况完整总结

| 情况 | 监视对象 | 写法 | 需要 deep |
|------|----------|------|-----------|
| 一（013） | ref 基本类型 | `watch(sum, cb)` | 不需要 |
| 二（014） | ref 对象类型 | `watch(person, cb, {deep})` | 需要 |
| 三（015） | reactive 对象 | `watch(person, cb)` | 不需要（默认深度） |
| 四（016） | 对象的某个属性 | `watch(() => person.name, cb)` | 属性是对象时需要 |
| 五（017） | 多个数据 | `watch([源1, 源2], cb, {deep})` | 看具体数据源 |

## 与后续课程的关系

017 是 watch 的最后一课。018 会讲 `watchEffect`——一种不需要手动指定监视数据的自动追踪方式。
