# 027 路由：to 的两种写法

## 本节目标

掌握 `<RouterLink>` 的 `to` 属性的两种写法：字符串和对象。

## 相对 026 的变化

App.vue 的导航区多了一个对象写法的 RouterLink：

```html
<!-- 字符串写法（简单跳转） -->
<RouterLink to="/news">新闻</RouterLink>

<!-- 对象写法（可以带参数） -->
<RouterLink :to="{ path: '/news', query: { id: 100, keyword: 'vue' } }">新闻详情</RouterLink>
```

## 核心知识点

### 写法一：字符串（简单跳转）

```html
<RouterLink to="/home">首页</RouterLink>
<RouterLink to="/news">新闻</RouterLink>
```

- `to` 前面**不加冒号**，值是字符串
- 适用于固定路径、不需要参数的场景

### 写法二：对象（带参数跳转）

```html
<RouterLink :to="{ path: '/news', query: { id: 100, keyword: 'vue' } }">
  新闻详情
</RouterLink>
```

- `:to` 前面**加冒号**（v-bind），值是对象
- 可以携带 `query` 参数（URL 中显示为 `?id=100&keyword=vue`）
- 也可以用 `name` 代替 `path`（028 会讲）

### 对象写法的完整结构

```ts
{
  name: 'xinwen',         // 命名路由名（可选，和 path 二选一）
  path: '/news',           // 路径（可选，和 name 二选一）
  query: { id: 100 },      // query 参数（可选）
  params: { id: 100 }      // params 参数（可选，031 会讲）
}
```

### 两种写法对比

| | 字符串写法 | 对象写法 |
|--|-----------|---------|
| 语法 | `to="/news"` | `:to="{ path: '/news', query: {...} }"` |
| 冒号 | 不加 | 加（v-bind） |
| 带参数 | ❌ 不方便 | ✅ 方便 |
| 适用场景 | 简单固定跳转 | 需要传参数、使用命名路由 |
