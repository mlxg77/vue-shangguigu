# 033 路由：replace 属性

## 本节目标

掌握 `<RouterLink>` 的 `replace` 属性——控制路由跳转是否留下历史记录。

## 相对 032 的变化

RouterLink 加了 `replace` 属性：

```html
<!-- 032 -->
<RouterLink to="/home">首页</RouterLink>

<!-- 033 -->
<RouterLink replace to="/home">首页</RouterLink>
```

## 核心知识点

### 1. 两种历史记录模式

| 模式 | 行为 | 浏览器后退按钮 |
|------|------|---------------|
| **push**（默认） | 把新页面**追加**到历史栈 | 逐步后退 |
| **replace** | 把当前页面**替换**为新页面 | 跳过当前页 |

### 2. 举例说明

假设访问顺序：A → B → C

**push 模式（默认）：**
```
历史栈：[A, B, C]
在 C 点后退 → 回到 B → 再后退 → 回到 A
```

**replace 模式（从 B 跳 C 时用 replace）：**
```
历史栈：[A, C]  ← B 被"顶掉"了
在 C 点后退 → 直接回到 A（跳过了 B）
```

### 3. 用法

```html
<RouterLink replace to="/home">首页</RouterLink>
<RouterLink replace :to="{ name: 'xinwen' }">新闻</RouterLink>
<RouterLink replace :to="{ path: '/about' }">关于</RouterLink>
```

直接在 RouterLink 上加 `replace` 即可，不需要赋值。

### 4. 适用场景

- **push**：需要能后退的场景（大多数情况）
- **replace**：不希望用户后退到某个页面的场景（如登录后跳转、重定向中间页）
