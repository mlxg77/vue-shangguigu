# 014 watch 监视：情况二

## 本节目标

监视 ref 定义的**对象类型**数据，理解 `deep` 深度监视的必要性。

## 相对 013 的变化

013 监视基本类型，直接传 ref 即可；014 监视对象类型，需要开启 `deep: true`。

## 核心知识点

### 1. 问题：默认不深度监视

```ts
let person = ref({ name: '张三', age: 18 })

// 不加 deep
watch(person, (newValue, oldValue) => {
  console.log('person变化了')
})
// 修改属性：person.value.name += '~'  → 不触发！
// 替换整个对象：person.value = { name: '李四' }  → 触发
```

**原因：** ref 监视对象时，默认比较的是 `.value` 的**引用地址**。
- 修改 `person.value.name` → 地址没变 → 不触发
- 替换 `person.value = {...}` → 地址变了 → 触发

### 2. 解决：开启 deep

```ts
watch(person, (newValue, oldValue) => {
  console.log('person变化了', newValue, oldValue)
}, { deep: true })
```

加了 `{ deep: true }` 后：
- 修改 `person.value.name` ✅ 触发
- 修改 `person.value.age` ✅ 触发
- 替换 `person.value = {...}` ✅ 触发

### 3. deep 模式下的 newValue/oldValue

```ts
// 修改 person.value.name = '李四'
watch(person, (newValue, oldValue) => {
  console.log(newValue)   // { name: '李四', age: 18 }
  console.log(oldValue)   // { name: '李四', age: 18 }  ← 注意！oldValue 也是新值！
}, { deep: true })
```

> 深度监视时，newValue 和 oldValue **指向同一个对象**（因为修改的是同一份 Proxy 内部数据），所以 oldValue 拿不到真正的旧值。

### 4. watch 三个参数完整说明

```ts
watch(
  person,                    // ① 被监视的数据
  (newValue, oldValue) => {},// ② 回调函数
  { deep: true, immediate: false }  // ③ 配置对象
)
```

| 配置项 | 作用 | 默认值 |
|--------|------|--------|
| `deep` | 深度监视对象内部属性变化 | false |
| `immediate` | 立即执行一次回调 | false |

## 情况一 vs 情况二

| | 情况一（013） | 情况二（014） |
|--|-------------|-------------|
| 数据类型 | ref 基本类型 | ref 对象类型 |
| 需要 deep | 不需要 | **需要** |
| oldValue | ✅ 正确 | ❌ 和 newValue 一样 |

## 与后续课程的关系

015 会讲情况三：监视 reactive 定义的对象类型——**默认就是深度监视**，不需要手动加 deep。
