# 040 storeToRefs

## 本节目标

用 `storeToRefs` 解构 store 的数据，解决直接解构丢失响应式的问题——类似 011 的 toRefs。

## 相对 039 的变化

```ts
// 039：每次都要写 countStore.xxx
<h2>当前求和为：{{ countStore.sum }}</h2>
<h3>欢迎来到:{{ countStore.school }}</h3>

// 040：解构后直接用
<h2>当前求和为：{{ sum }}</h2>
<h3>欢迎来到:{{ school }}</h3>
```

## 核心知识点

### 1. 问题：直接解构丢失响应式

```ts
const countStore = useCountStore()
const { sum, school, address } = countStore  // ❌ 丢失响应式！
```

> 和 011 的 reactive 解构问题一样——解构出来的是普通值，不再是响应式。

### 2. 解决：storeToRefs

```ts
import { storeToRefs } from 'pinia'

const countStore = useCountStore()
const { sum, school, address } = storeToRefs(countStore)  // ✅ 保持响应式
```

`storeToRefs` 会把 store 的 state 和 getters 转成 ref，解构后仍然响应式。

### 3. storeToRefs 只转数据，不转方法

```ts
const { sum, school, address } = storeToRefs(countStore)  // ✅ 数据
// const { increment } = storeToRefs(countStore)          // ❌ 方法不会转

// 方法直接从 store 调用
countStore.increment(n.value)  // ✅
```

> **为什么方法不转？** 因为方法不需要响应式包装，直接通过 `store.方法名()` 调用即可。

### 4. 模板中使用

```html
<!-- 数据：解构后直接用 -->
<h2>当前求和为：{{ sum }}</h2>
<h3>欢迎来到:{{ school }}，坐落于：{{ address }}</h3>

<!-- 方法：通过 store 调用 -->
<button @click="add">加</button>
```

```ts
function add() {
  countStore.increment(n.value)  // 方法还是通过 store 调
}
function minus() {
  countStore.sum -= n.value      // 也可以直接改 store 的属性
}
```

### 5. LoveTalk 中的使用

```ts
const talkStore = useTalkStore()
const { talkList } = storeToRefs(talkStore)  // ✅ 数据解构

function getLoveTalk() {
  talkStore.getATalk()  // 方法通过 store 调
}
```

### 6. storeToRefs vs toRefs

| | toRefs（011） | storeToRefs（040） |
|--|--------------|-------------------|
| 来源 | Vue 核心包 | Pinia 包 |
| 作用对象 | reactive 对象 | Pinia store |
| 转什么 | 所有属性 | **只转 state + getters**（不转 actions） |
