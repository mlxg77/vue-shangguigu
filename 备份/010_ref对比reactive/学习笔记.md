# 010 ref 对比 reactive

## 本节目标

正式对比 `ref` 和 `reactive` 的区别，揭示 reactive 的"整体重新赋值"陷阱，给出使用建议。

## 相对 009 的变化

把 ref 和 reactive 放在同一个组件里对比，特别展示了 reactive **不能整体替换**的问题：

```ts
let car = reactive({ brand: '奔驰', price: 100 })  // reactive
let sum = ref(0)                                     // ref

function changeCar() {
  // car = { brand: '奥拓', price: 1 }           // ❌ 页面不更新
  // car = reactive({ brand: '奥拓', price: 1 })  // ❌ 页面不更新
  Object.assign(car, { brand: '奥拓', price: 1 })  // ✅ 页面更新
}
```

## 核心知识点

### 1. reactive 不能整体重新赋值

```ts
let car = reactive({ brand: '奔驰', price: 100 })

function changeCar() {
  car = { brand: '奥拓', price: 1 }  // ❌ 页面不更新
}
```

**原因：** `car` 是一个 `let` 变量指向 reactive 返回的 Proxy 对象。直接 `car = 新对象` 只是让变量指向了一个新的普通对象，**原 Proxy 对象没变**，Vue 检测不到变化。

**正确做法：** 用 `Object.assign` 把新属性合并进原 Proxy 对象：

```ts
Object.assign(car, { brand: '奥拓', price: 1 })  // ✅ 修改原 Proxy，页面更新
```

### 2. ref 没有这个问题

```ts
let sum = ref(0)

function changeSum() {
  sum.value += 1         // ✅
  // sum.value = 999     // ✅ 也可以整体替换
}
```

ref 的 `.value` 重新赋值会触发 setter，Vue 能检测到。

### 3. 完整对比表

| 对比项 | ref | reactive |
|--------|-----|----------|
| 支持基本类型 | ✅ | ❌ |
| 支持对象类型 | ✅ | ✅ |
| JS 中访问 | `xxx.value.属性` | `xxx.属性` |
| 模板中 | 自动解包，直接 `xxx` | 直接 `xxx.属性` |
| 修改单个属性 | `car.value.price = 200` | `car.price = 200` |
| **整体替换** | `car.value = newObj` ✅ | `car = newObj` ❌ |
| 整体替换方案 | 直接 `.value =` | `Object.assign(car, newObj)` |

### 4. 使用建议

- **基本类型**：只能用 `ref`
- **对象类型**：推荐用 `ref`（因为可以整体替换，更灵活）
- 如果用 `reactive`，需要整体替换时用 `Object.assign`

> 很多团队的实际规范是：**统一用 ref**，简单省心。reactive 在某些场景（如表单对象）也有用，但要注意赋值陷阱。
