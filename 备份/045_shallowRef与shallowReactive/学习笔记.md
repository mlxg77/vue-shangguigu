# 045 shallowRef 与 shallowReactive

## 本节目标

掌握 `shallowRef` 和 `shallowReactive`——只做**浅层响应式**的 API，理解它们与 `ref`/`reactive` 的区别。

## 核心知识点

### 1. shallowRef

`shallowRef` 只对 `.value` 的替换做出响应，**不对对象内部属性变化做出响应**。

```ts
let sum = shallowRef(0)          // 基本类型——和 ref 一样
let person = shallowRef({        // 对象类型——只有整体替换才响应
  name: '张三',
  age: 18
})

function changeName() {
  person.value.name = '李四'     // ❌ 不会触发视图更新
}

function changeAge() {
  person.value.age += 1          // ❌ 不会触发视图更新
}

function changePerson() {
  person.value = { name: 'tony', age: 100 }  // ✅ 触发视图更新（整体替换）
}
```

> 对比 ref：`ref` 包裹对象时会用 `reactive` 处理内部对象，所以修改属性能响应。`shallowRef` 不会做这层处理。

### 2. shallowReactive

`shallowReactive` 只对**第一层属性**做响应式，深层属性不响应。

```ts
let car = shallowReactive({
  brand: '奔驰',
  options: {
    color: '红色',
    engine: 'V8'
  }
})

function changeBrand() {
  car.brand = '宝马'             // ✅ 第一层属性，触发响应
}

function changeColor() {
  car.options.color = '紫色'     // ❌ 第二层属性，不触发
}

function changeEngine() {
  car.options.engine = 'V12'     // ❌ 第二层属性，不触发
}
```

> 对比 reactive：`reactive` 会深度代理所有层级，修改任何深度的属性都能响应。

### 3. 对比总结

| API | 响应层级 | 适合场景 |
|-----|---------|---------|
| `ref` | 深层（对象经 reactive 处理） | 需要修改对象内部属性 |
| `shallowRef` | 仅 `.value` 替换 | 只会整体替换对象，不修改内部属性 |
| `reactive` | 深层（所有属性） | 需要修改深层属性 |
| `shallowReactive` | 仅第一层属性 | 只有第一层属性会变，深层属性不变 |

### 4. 为什么需要浅层响应式？

- **性能优化**：深层响应式需要递归代理对象所有属性，大对象时开销大
- **避免不必要的代理**：有些对象（如第三方库实例）不需要被代理
- **场景**：后端返回的大数据结构、只需要整体替换的数据
