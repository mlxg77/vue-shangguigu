# 023 自定义 hooks

## 本节目标

把组件中的可复用逻辑抽成独立的 `useXxx` 函数——Vue 3 的 Composition API 代码复用方案。

## 核心知识点

### 1. 什么是自定义 hook

自定义 hook 就是一个普通函数，命名以 `use` 开头，内部使用 Composition API（ref、reactive、computed、onMounted 等），**返回组件需要的数据和方法**。

### 2. useSum —— 求和逻辑

```ts
// hooks/useSum.ts
import { ref, onMounted, computed } from 'vue'

export default function () {
  let sum = ref(0)
  let bigSum = computed(() => sum.value * 10)  // 计算属性
  function add() { sum.value += 1 }
  onMounted(() => { add() })  // 生命周期钩子也能写在 hook 里

  return { sum, add, bigSum }
}
```

### 3. useDog —— 狗图逻辑

```ts
// hooks/useDog.ts
import { reactive, onMounted } from 'vue'
import axios from 'axios'

export default function () {
  let dogList = reactive(['https://...'])
  async function getDog() {
    let result = await axios.get('https://dog.ceo/api/breed/pembroke/images/random')
    dogList.push(result.data.message)
  }
  onMounted(() => { getDog() })

  return { dogList, getDog }
}
```

### 4. 组件中使用 hooks

```ts
// Person.vue
import useSum from '../hooks/useSum'
import useDog from '../hooks/useDog'

const { sum, add, bigSum } = useSum()
const { dogList, getDog } = useDog()
```

组件变得**非常干净**——只有 import + 解构，所有逻辑都在 hooks 里。

### 5. 自定义 hook 的优势

| 对比 | 不用 hook | 用 hook |
|------|----------|---------|
| 逻辑位置 | 全堆在组件里 | 按功能拆分到独立文件 |
| 复用性 | 复制粘贴 | 多个组件 import 同一个 hook |
| 可维护性 | 改一处要找组件 | 改 hook 文件即可 |

### 6. hook 里可以用什么

- ✅ `ref` / `reactive` — 响应式数据
- ✅ `computed` — 计算属性
- ✅ `watch` / `watchEffect` — 监视
- ✅ `onMounted` 等生命周期 — 会绑定到调用 hook 的组件上
- ✅ 发请求（axios 等）

## 与 Vue 2 mixin 的对比

Vue 2 用 mixin 复用逻辑，但有命名冲突、来源不清晰等问题。Vue 3 的自定义 hook：
- 来源清晰（import 就能看到）
- 不会命名冲突（解构时可以重命名）
- 类型推导更好
