# 026 路由：路由器工作模式

## 本节目标

理解 Vue Router 的两种工作模式：`createWebHistory`（history 模式）和 `createWebHashHistory`（hash 模式）。

## 相对 025 的变化

```ts
// 025 用的是 history 模式
history: createWebHistory()

// 026 改成了 hash 模式
history: createWebHashHistory()
```

## 核心知识点

### 两种模式对比

| 特性 | createWebHistory | createWebHashHistory |
|------|-----------------|---------------------|
| URL 格式 | `/about` | `/#/about` |
| 底层原理 | HTML5 History API | URL hash（`#`） |
| 需要服务器配置 | **需要**（否则刷新 404） | 不需要 |
| 浏览器前进/后退 | 支持 | 支持 |
| SEO 友好 | ✅ 更好 | ❌ 较差 |

### history 模式

```
URL: http://localhost:5173/about
```

- 利用 HTML5 的 `pushState` API 修改 URL，不刷新页面
- **刷新时浏览器会向服务器请求 `/about`**，如果服务器没配置对应路由，会返回 404
- 需要**服务器配置**：把所有路由都重定向到 `index.html`（让 Vue Router 在前端处理）

### hash 模式

```
URL: http://localhost:5173/#/about
```

- `#` 后面的部分（hash）变化不会触发服务器请求
- 刷新时服务器只请求 `/`，返回 `index.html`，前端路由处理 `#/about`
- **不需要服务器配置**，但 URL 不好看

### 如何选择

| 场景 | 推荐 |
|------|------|
| 正式项目（有后端配合） | history 模式（URL 干净、SEO 好） |
| 学习/演示/无后端 | hash 模式（省事，不会 404） |

> 本课程后续大多使用 `createWebHistory`（因为 Vite 开发服务器默认支持 history 模式刷新）。
