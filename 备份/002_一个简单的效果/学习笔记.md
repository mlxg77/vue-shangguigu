# 002 一个简单的效果

## 本节目标

在 001 的基础上引入子组件 `Person`，实现数据展示和按钮交互——经典的 Options API 写法。

## 相对 001 的变化

| 变化点 | 001 | 002 |
|--------|-----|-----|
| 子组件 | 无 | 新增 `components/Person.vue` |
| App.vue | 只有一个 h1 | 引入并注册 Person，模板里用 `<Person/>` |
| 数据与交互 | 无 | Person 有 data、methods，有按钮事件 |

## 核心知识点

### 1. 父组件引入子组件的步骤（Options API 写法）

```ts
// App.vue
import Person from './components/Person.vue' // ① 引入
export default {
  name: 'App',
  components: { Person } // ② 注册
}
```
```html
<!-- ③ 使用，<Person/> 等价于 <Person></Person> -->
<Person/>
```

> 三步缺一不可：引入 → 注册 → 使用。

### 2. Options API 的 data 和 methods

Person.vue 展示了 Vue 2 风格的写法：

```ts
export default {
  name: 'Person',
  data() {
    return {
      name: '张三',   // 数据
      age: 18,
      tel: '13888888888'
    }
  },
  methods: {
    changeName() { this.name = 'zhang-san' },  // 用 this 访问数据
    changeAge() { this.age += 1 },
    showTel() { alert(this.tel) }
  }
}
```

**特点：**
- `data()` 返回的对象就是响应式数据
- `methods` 里的方法通过 `this.xxx` 访问 data 中的数据
- 模板里 `{{ name }}` 和 `@click="changeName"` 直接用，不需要 this

### 3. 模板语法

| 语法 | 作用 | 示例 |
|------|------|------|
| `{{ }}` | 插值（显示数据） | `{{ name }}` |
| `@click` | 事件绑定 | `@click="changeName"` |
| `<style scoped>` | 样式只作用于当前组件 | 防止样式泄漏到其他组件 |

## 与后续课程的关系

002 是**纯 Options API 写法**（data + methods + this）。003 开始引入 `setup` 函数，数据不再写在 data 里、方法不再写在 methods 里、`this` 也被弱化了。
