# 036 准备一个效果

## 本节目标

搭建一个"求和器 + 土味情话"的效果，数据全在组件内部——为 Pinia 学习做铺垫（暴露问题）。

## 核心知识点

### 两个组件各管各的数据

**Count.vue** — 求和器
```ts
let sum = ref(1)   // 求和数据（组件内部）
let n = ref(1)     // 用户选择的数字
function add() { sum.value += n.value }
function minus() { sum.value -= n.value }
```

**LoveTalk.vue** — 土味情话
```ts
let talkList = reactive([...])  // 情话列表（组件内部）
async function getLoveTalk() {
  let { data: { content: title } } = await axios.get('...')
  talkList.unshift({ id: nanoid(), title })
}
```

### 暴露的问题（Pinia 要解决的）

- `sum` 和 `talkList` 都锁在各自组件里
- 如果 LoveTalk 想知道 sum 的值——拿不到
- 如果其他组件也想显示情话列表——拿不到
- 组件销毁后数据丢失

> 036 就是"没有状态管理"的典型场景，为 037-043 的 Pinia 学习铺垫。
