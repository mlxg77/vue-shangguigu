# 009 ref 创建：对象类型的响应式数据

## 本节目标

展示 `ref` 也能处理对象类型——内部会自动调用 `reactive`。

## 相对 008 的变化

同样是操作 car 和 games，但改用 `ref` 而非 `reactive`：

```ts
// 008（reactive）
let car = reactive({ brand: '奔驰', price: 100 })
car.price += 10  // 直接访问

// 009（ref）
let car = ref({ brand: '奔驰', price: 100 })
car.value.price += 10  // 需要 .value
```

## 核心知识点

### 1. ref 用于对象类型

```ts
import { ref } from 'vue'

let car = ref({ brand: '奔驰', price: 100 })
let games = ref([
  { id: '01', name: '王者荣耀' },
  { id: '02', name: '原神' },
  { id: '03', name: '三国志' }
])
```

### 2. JS 中需要 `.value`

```ts
function changePrice() {
  car.value.price += 10        // 先 .value 拿到对象，再访问属性
}
function changeFirstGame() {
  games.value[0].name = '流星蝴蝶剑'
}
```

### 3. 模板中不需要 `.value`

```html
<h2>汽车信息：一辆{{ car.brand }}车，价值{{ car.price }}万</h2>
```

### 4. ref 处理对象的内部原理

当 `ref()` 收到**对象**时，内部会自动调用 `reactive()` 把对象包成 Proxy：

```
ref({ brand: '奔驰', price: 100 })
  → RefImpl {
      value: reactive({ brand: '奔驰', price: 100 })  ← 内部用 reactive
    }
```

所以 `ref` 既能处理基本类型（直接包装），也能处理对象类型（内部委托给 reactive）。

### 5. ref 和 reactive 可以混用

```ts
let car = ref({ brand: '奔驰', price: 100 })  // ref 处理对象
let obj = reactive({ x: 999 })                 // reactive 处理对象
```

## 008 vs 009 对比

| 对比项 | 008（reactive） | 009（ref） |
|--------|----------------|-----------|
| 声明 | `reactive({...})` | `ref({...})` |
| JS 访问 | `car.price` | `car.value.price` |
| 模板访问 | `car.price` | `car.price` |
| 重新赋值 | ❌ `car = {...}` 不更新 | ✅ `car.value = {...}` 更新（010 会讲） |
