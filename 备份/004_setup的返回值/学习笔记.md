# 004 setup 的返回值

## 本节目标

补充 setup 返回值的两种形式：返回对象（常用）和返回渲染函数（了解）。

## 相对 003 的变化

代码几乎和 003 完全一致，唯一新增的是注释中揭示了 setup 返回值的第二种写法：

```ts
setup() {
  let name = '张三'
  // ...
  return { name, age, tel, changeName, changeAge, showTel } // 写法一：返回对象（常用）

  // setup的返回值也可以是一个渲染函数
  // return () => '哈哈'  // 写法二：返回渲染函数（了解即可）
}
```

## 核心知识点

### setup 返回值的两种形式

| 返回类型 | 作用 | 使用场景 |
|----------|------|----------|
| **对象** `{ name, age, ... }` | 对象中的属性和方法可直接在模板中使用 | 99% 的场景都用这种 |
| **渲染函数** `() => '哈哈'` | 完全替代模板，用函数返回虚拟 DOM | 需要完全编程式控制渲染时（很少用） |

### 返回对象时

```ts
return { name, age, tel, changeName, changeAge, showTel }
```
→ 模板里可以 `{{ name }}`、`@click="changeName"`。

### 返回渲染函数时

```ts
return () => '哈哈'
```
→ 页面上不显示 template 里的内容，直接显示"哈哈"（整个模板被忽略）。

> 渲染函数的深入用法不在本课程范围内，这里只需知道有这种写法即可。

## 与 003 的关系

004 是 003 的**纯补充**，没有功能变化，只是多讲了"返回值也可以是渲染函数"这一个知识点。实际开发中几乎都用返回对象的形式。
