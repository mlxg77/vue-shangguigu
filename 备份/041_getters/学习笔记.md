# 041 getters

## 本节目标

掌握 Pinia store 中的 `getters`——相当于 store 的计算属性。

## 相对 040 的变化

store 的 count.ts 中新增了 `getters` 配置：

```ts
getters: {
  bigSum: state => state.sum * 10,
  upperSchool(): string {
    return this.school.toUpperCase()
  }
}
```

组件中通过 storeToRefs 解构 getters 使用。

## 核心知识点

### 1. getters 的两种写法

**写法一：箭头函数（参数是 state）**
```ts
getters: {
  bigSum: state => state.sum * 10
}
```
- 用 `state.xxx` 访问数据
- 不能用 `this`（箭头函数没有 this）

**写法二：普通函数（用 this）**
```ts
getters: {
  upperSchool(): string {
    return this.school.toUpperCase()
  }
}
```
- 用 `this.xxx` 访问数据和其他 getter
- 可以调用其他 getters：`this.bigSum`

### 2. getters 的特点

- **有缓存**：依赖没变时不会重新计算（和 computed 一样）
- **响应式**：依赖的 state 变化时自动更新
- **只读**：不能在 getters 里修改 state

### 3. 组件中使用 getters

```ts
const { sum, school, address, bigSum, upperSchool } = storeToRefs(countStore)
```

```html
<h2>当前求和为：{{ sum }}，放大10倍后：{{ bigSum }}</h2>
<h3>欢迎来到:{{ school }}，大写：{{ upperSchool }}</h3>
```

> storeToRefs 会把 getters 也转成 ref，解构后仍然响应式。

### 4. getters vs computed

| | computed（012） | getters（041） |
|--|----------------|---------------|
| 定义位置 | 组件中 | store 中 |
| 依赖 | 组件内的 ref/reactive | store 的 state |
| 使用范围 | 当前组件 | 所有用到该 store 的组件 |
| 缓存 | ✅ | ✅ |
