# 034 路由：编程式路由导航

## 本节目标

用 JS 代码实现路由跳转（不依赖 `<RouterLink>`），掌握 `useRouter()` 的用法。

## 相对 033 的变化

News.vue 中除了声明式的 RouterLink，还新增了一个按钮用 JS 代码跳转：

```html
<!-- 声明式导航（RouterLink） -->
<RouterLink :to="{ name: 'xiang', query: {...} }">{{ news.title }}</RouterLink>

<!-- 编程式导航（按钮 + JS） -->
<button @click="showNewsDetail(news)">查看新闻</button>
```

## 核心知识点

### 1. useRouter 获取路由器

```ts
import { useRouter } from 'vue-router'
const router = useRouter()
```

| API | 作用 |
|-----|------|
| `useRoute()` | 获取当前路由信息（path、query、params）— 只读 |
| `useRouter()` | 获取路由器实例（push、replace）— 用来跳转 |

### 2. router.push 和 router.replace

```ts
function showNewsDetail(news: NewsInter) {
  router.replace({
    name: 'xiang',
    query: {
      id: news.id,
      title: news.title,
      content: news.content
    }
  })
}
```

| 方法 | 行为 | 对应 RouterLink |
|------|------|----------------|
| `router.push(...)` | 留下历史记录 | `<RouterLink to="...">` |
| `router.replace(...)` | 不留历史记录 | `<RouterLink replace to="...">` |

### 3. 声明式 vs 编程式

| | 声明式导航 | 编程式导航 |
|--|-----------|-----------|
| 写法 | `<RouterLink to="...">` | `router.push(...)` |
| 触发方式 | 点击链接 | 任意 JS 代码（按钮、定时器、条件判断后） |
| 灵活性 | 较低 | 高（可以先执行逻辑再跳转） |
| 适用场景 | 简单的页面切换 | 需要条件判断/异步操作后跳转 |

### 4. 编程式导航的典型场景

```ts
// 登录成功后跳转
async function login() {
  const res = await api.login(...)
  if (res.success) {
    router.push({ name: 'home' })  // 登录成功才跳
  }
}

// 定时跳转
setTimeout(() => {
  router.push({ name: 'home' })
}, 3000)

// 带参数跳转
router.push({
  name: 'detail',
  query: { id: 1 }
})
```
