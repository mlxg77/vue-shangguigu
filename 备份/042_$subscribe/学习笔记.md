# 042 $subscribe

## 本节目标

掌握 Pinia 的 `$subscribe`——监听 store 状态变化，实现数据持久化等需求。

## 相对 041 的变化

LoveTalk.vue 中新增了 $subscribe，监听 talkStore 变化并将数据存到 localStorage：

```ts
talkStore.$subscribe((mutate, state) => {
  console.log('talkStore里面保存的数据发生了变化', mutate, state)
  localStorage.setItem('talkList', JSON.stringify(state.talkList))
})
```

## 核心知识点

### 1. $subscribe 的语法

```ts
store.$subscribe((mutation, state) => {
  // mutation: 变更信息（类型、storeId 等）
  // state: 最新的 state 对象
}, options)  // options 可选
```

| 参数 | 作用 |
|------|------|
| mutation | 变更详情（事件类型 'direct'/'patch object'/'patch function'） |
| state | 变更后的最新 state |

### 2. 典型场景：数据持久化

```ts
talkStore.$subscribe((mutate, state) => {
  localStorage.setItem('talkList', JSON.stringify(state.talkList))
})
```

每次 talkStore 数据变化时，自动把 talkList 存到 localStorage。

配合 store 初始化时从 localStorage 读取：
```ts
state() {
  return {
    talkList: JSON.parse(localStorage.getItem('talkList') as string) || []
  }
}
```

→ 实现了**刷新页面数据不丢失**。

### 3. $subscribe vs watch

| | watch（013-017） | $subscribe（042） |
|--|-----------------|------------------|
| 监听对象 | ref / reactive | Pinia store |
| 获取旧值 | ✅ oldValue | ❌ 只有新 state |
| 粒度 | 精确到某个属性 | 整个 store 任意变化都触发 |
| 适用场景 | 组件内响应式数据 | store 级别的副作用 |

### 4. $subscribe 的特点

- store 中**任何**属性变化都会触发（不像 watch 需要指定监听谁）
- 回调中拿到的是**整个 state**，不是某个属性的新旧值
- 适合做"store 一变就执行某事"的场景
