# 018 watchEffect

## 本节目标

掌握 `watchEffect`——自动追踪依赖的监视方式，对比 `watch` 的区别。

## 核心知识点

### 1. 需求场景

"当水温 ≥ 60 度或水位 ≥ 80cm 时，给服务器发请求"——涉及两个数据（temp、height）。

### 2. watch 实现（对比）

```ts
watch([temp, height], (newValue, oldValue) => {
  let [newTemp, newHeight] = newValue
  if (newTemp >= 60 || newHeight >= 80) {
    console.log('给服务器发请求')
  }
})
```

**缺点：**
- 必须手动列出所有要监视的数据 `[temp, height]`
- 需要解构 newValue 才能拿到具体值
- 不会立即执行（要等数据变化才触发）

### 3. watchEffect 实现

```ts
watchEffect(() => {
  // 函数体里用到的响应式数据，自动成为依赖
  if (temp.value >= 60 || height.value >= 80) {
    console.log('给服务器发请求')
  }
})
```

**特点：**
- 不需要指定监视谁——**用到谁就监视谁**
- **立即执行一次**（组件加载时就跑一遍）
- 后续 temp 或 height 变化时自动重新执行

### 4. watch vs watchEffect

| 对比项 | watch | watchEffect |
|--------|-------|-------------|
| 指定监视数据 | 必须手动指定 | 自动追踪（用到谁监谁） |
| 初始执行 | 不执行（除非 immediate） | **立即执行一次** |
| 获取 oldValue | ✅ 有 | ❌ 没有 |
| 适用场景 | 需要新旧值对比 | 只关心"当前状态是否满足条件" |

### 5. watchEffect 的依赖收集原理

```ts
watchEffect(() => {
  if (temp.value >= 60 || height.value >= 80) {
    // ↑ 这里读了 temp.value 和 height.value
    // → Vue 自动把它们标记为依赖
    console.log('给服务器发请求')
  }
})
```

和 computed 类似，watchEffect 内部读取了哪些 ref/reactive，就自动追踪哪些。区别是 computed 返回值，watchEffect 执行副作用。

## 使用建议

- 需要 oldValue → 用 watch
- 需要精确控制监听哪些数据 → 用 watch
- 只关心"条件满足就执行" → 用 watchEffect（更简洁）
- 依赖多且不确定 → 用 watchEffect（自动追踪）
