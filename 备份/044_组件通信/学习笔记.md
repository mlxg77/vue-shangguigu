# 044 组件通信

## 本节目标

系统学习 Vue 3 中 **9 种组件通信方式**，理解每种方式的适用场景和代码写法。

## 项目结构

本节用路由将 9 种通信方式组织为 9 个页面，每个页面独立演示一种方式：

```

│   ├── 02_custom-event/   # 子→父：自定义事件
│   ├── 03_mitt/           # 任意组件：mitt（事件总线）
│   ├── 04_v-model/        # 父↔子：v-model
```

## 1. props（父 → 子）

**方向**：父组件向子组件传递数据和方法。

```html
<!-- Father.vue -->
<Child :car="car" :sendToy="getToy"/>
```

```html
<!-- Child.vue -->
defineProps(['car','sendToy'])
<!-- 子组件也可以通过调用父传来的方法，间接实现 子→父 通信 -->
<button @click="sendToy(toy)">把玩具给父亲</button>
```

> props 不仅可以传数据，也可以传函数（子组件调用函数把数据传回父组件）。

---

## 2. 自定义事件（子 → 父）

**方向**：子组件向父组件传递数据。

```html
<!-- Father.vue -->
<Child @send-toy="saveToy"/>
```

```ts
// Child.vue
const emit = defineEmits(['send-toy'])
emit('send-toy', toy)
```

> 与 props 传函数的区别：自定义事件语义更清晰，父组件只管"监听"，子组件只管"触发"。

---

## 3. mitt（任意组件间）

**方向**：任意两个组件之间通信（不限于父子关系）。

```ts
// utils/emitter.ts
import mitt from 'mitt'
const emitter = mitt()
export default emitter
```

```ts
// Child1.vue（发送方）
import emitter from '@/utils/emitter'
emitter.emit('send-toy', toy)

// Child2.vue（接收方）
emitter.on('send-toy', (value) => { toy.value = value })

// 组件卸载时解绑
onUnmounted(() => {
  emitter.off('send-toy')
})
```

> Vue 3 移除了 Vue 2 的 `$bus`（`$on`/`$emit`/`$off`），需要用第三方库 `mitt` 替代。

---

## 4. v-model（父 ↔ 子）

**方向**：父子组件双向绑定。

```html
<!-- Father.vue -->
<!-- 重命名 v-model（不用默认的 modelValue） -->
<AtguiguInput v-model:ming="username" v-model:mima="password"/>
```

```html
<!-- AtguiguInput.vue -->
<input :value="ming" @input="emit('update:ming', $event.target.value)">
<input :value="mima" @input="emit('update:mima', $event.target.value)">

defineProps(['ming','mima'])
const emit = defineEmits(['update:ming','update:mima'])
```

> `v-model:ming="username"` 等价于 `:ming="username" @update:ming="username = $event"`。
> 一个组件上可以使用多个 v-model（通过不同名称区分）。

---

## 5. $attrs（祖 → 孙）

**方向**：父组件向孙子组件传递数据（跨层级）。

```html
<!-- Father.vue -->
<Child :a="a" :b="b" :c="c" :d="d" :updateA="updateA"/>
```

```html
<!-- Child.vue（中间层，不需要声明 props，直接透传） -->
<GrandChild v-bind="$attrs"/>
```

```html
<!-- GrandChild.vue（接收） -->
defineProps(['a','b','c','d','updateA'])
```

> `$attrs` 包含父组件传过来但子组件**没有用 props 接收**的所有属性和方法。
> 子组件用 `v-bind="$attrs"` 把它们继续传给孙组件。

---

## 6. $refs / $parent（父 ↔ 子直接访问）

**方向**：父通过 `$refs` 访问子组件，子通过 `$parent` 访问父组件。

```html
<!-- Father.vue -->
<Child1 ref="c1"/>
<Child2 ref="c2"/>

<button @click="changeToy">修改Child1的玩具</button>
```

```ts
// Father.vue
let c1 = ref()
function changeToy() {
  c1.value.toy = '小猪佩奇'  // 直接修改子组件数据
}

// 父组件需要 defineExpose 暴露数据
defineExpose({ house })
```

```ts
// Child1.vue
function minusHouse(parent) {
  parent.house -= 1  // 通过 $parent 修改父组件数据
}
defineExpose({ toy, book })  // 必须暴露，否则父组件访问不到
```

> 注意：`<script setup>` 默认是封闭的，必须用 `defineExpose` 暴露数据，父/子组件才能通过 `$refs`/`$parent` 访问到。

---

## 7. provide / inject（祖 → 后代）

**方向**：祖先组件向所有后代组件提供数据（不限层级）。

```ts
// Father.vue（祖先）
import { provide } from 'vue'
let money = ref(100)
function updateMoney(value) { money.value -= value }

provide('moneyContext', { money, updateMoney })
provide('car', car)
```

```ts
// GrandChild.vue（后代，中间层 Child.vue 不需要写任何代码）
import { inject } from 'vue'
let { money, updateMoney } = inject('moneyContext', { money: 0, updateMoney: () => {} })
let car = inject('car', { brand: '未知', price: 0 })
```

> `inject` 第二个参数是默认值（当祖先没有 provide 时使用）。
> 与 `$attrs` 的区别：`$attrs` 需要中间层手动透传，`provide/inject` 中间层完全不需要感知。

---

## 8. Pinia（任意组件间）

**方向**：任意组件间共享状态。

直接参考 036-043 的 Pinia 章节。Pinia 是最推荐的任意组件通信方案：
- 数据集中管理在 store
- 任何组件都能读、都能改
- 自带响应式
- 支持数据持久化

## 9. 插槽（父 → 子模板传递）

### 9.1 默认插槽

父组件决定子组件内部显示什么内容：

```html
<!-- Father.vue -->
<Category title="热门游戏列表">
  <ul>
    <li v-for="g in games" :key="g.id">{{ g.name }}</li>
  </ul>
</Category>
```

```html
<!-- Category.vue -->
<div class="category">
  <h2>{{ title }}</h2>
  <slot>默认内容</slot>  <!-- 父传的内容在这里渲染 -->
</div>
```

### 9.2 具名插槽

一个组件有多个插槽位置，用名称区分：

```html
<!-- Category.vue -->
<slot name="s1">默认内容1</slot>
<slot name="s2">默认内容2</slot>
```

```html
<!-- Father.vue -->
<Category>
  <template #s1>
    <h2>热门游戏列表</h2>
  </template>
  <template #s2>
    <ul>...</ul>
  </template>
</Category>
```

> `v-slot:s1` 简写为 `#s1`。

### 9.3 作用域插槽

**方向**：子组件把数据传给父组件，父组件决定如何渲染。

```html
<!-- Game.vue（子组件） -->
<slot :youxi="games" x="哈哈" y="你好"></slot>
```

```html
<!-- Father.vue（父组件） -->
<Game>
  <template #default="{ youxi }">
    <h3 v-for="g in youxi" :key="g.id">{{ g.name }}</h3>
  </template>
</Game>
```

> 作用域插槽的本质：**数据在子组件，渲染结构由父组件决定**。
> 适合"数据相同但展示形式不同"的场景（如同一组游戏数据，可以用 ul、ol、h3 三种方式展示）。

---

## 9 种通信方式总结

| 方式 | 方向 | 关键 API |
|------|------|----------|
| props | 父→子 | `defineProps` |
| 自定义事件 | 子→父 | `defineEmits` + `@事件名` |
| mitt | 任意 | `emitter.emit` / `emitter.on` |
| v-model | 父↔子 | `v-model:名称` + `update:名称` |
| $attrs | 祖→孙 | `$attrs` + `v-bind` |
| $refs/$parent | 父↔子 | `$refs` / `$parent` + `defineExpose` |
| provide/inject | 祖→后代 | `provide` / `inject` |
| Pinia | 任意 | `defineStore` / `storeToRefs` |
| 插槽 | 父→子（默认/具名）/ 子→父（作用域） | `<slot>` / `#名称` |
