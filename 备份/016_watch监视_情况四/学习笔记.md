# 016 watch 监视：情况四

## 本节目标

监视响应式对象中的**某个属性**——需要用函数式写法 `() => obj.prop`。

## 相对 015 的变化

015 监视整个对象；016 只监视对象中的**某一个属性**（如 `person.name` 或 `person.car`）。

## 核心知识点

### 1. 问题：直接写属性名不行

```ts
let person = reactive({ name: '张三', age: 18, car: { c1: '奔驰', c2: '宝马' } })

// ❌ 错误写法：直接传 person.name
watch(person.name, (newValue, oldValue) => {
  // person.name 是一个普通值（'张三'），不是响应式数据，watch 无法监视
})
```

### 2. 解决：写成 getter 函数

```ts
// ✅ 监视基本类型属性
watch(() => person.name, (newValue, oldValue) => {
  console.log('person.name变化了', newValue, oldValue)
})
```

`() => person.name` 是一个 getter 函数，watch 会自动追踪它的返回值变化。

### 3. 监视对象类型属性（需要 deep）

```ts
// ✅ 监视对象类型属性
watch(() => person.car, (newValue, oldValue) => {
  console.log('person.car变化了', newValue, oldValue)
}, { deep: true })
```

**为什么还需要 deep？** 因为 `() => person.car` 返回的是 car 对象的引用。修改 `person.car.c1` 时引用没变，需要 deep 才能检测到内部变化。

### 4. 各种操作是否触发

| 操作 | 触发？ | 原因 |
|------|--------|------|
| `person.car.c1 = '奥迪'` | ✅ | deep 监视到内部属性变化 |
| `person.car = { c1: '雅迪', c2: '爱玛' }` | ✅ | getter 返回值变了 |

## 情况四要点

| 属性类型 | 写法 | 需要 deep |
|----------|------|-----------|
| 基本类型（name） | `() => person.name` | 不需要 |
| 对象类型（car） | `() => person.car` | **需要** |
