# 015 watch 监视：情况三

## 本节目标

监视 reactive 定义的**对象类型**数据——理解它和 ref 对象类型的区别（默认深度监视）。

## 相对 014 的变化

| | 014（ref 对象） | 015（reactive 对象） |
|--|----------------|---------------------|
| 数据定义 | `ref({...})` | `reactive({...})` |
| 默认深度监视 | ❌ 需要手动加 `deep:true` | ✅ **默认就是深度监视** |
| oldValue | ❌ 和 newValue 相同 | ❌ 和 newValue 相同 |

## 核心知识点

### 1. reactive 默认深度监视

```ts
let person = reactive({ name: '张三', age: 18 })

// 不需要 deep: true！
watch(person, (newValue, oldValue) => {
  console.log('person变化了', newValue, oldValue)
})
```

修改任何层级的属性都会触发：
- `person.name += '~'` ✅ 触发
- `person.age += 1` ✅ 触发
- `Object.assign(person, { name: '李四' })` ✅ 触发

### 2. 嵌套对象也是自动深度的

```ts
let obj = reactive({ a: { b: { c: 666 } } })

watch(obj, (newValue, oldValue) => {
  console.log('Obj变化了', newValue, oldValue)
})

function test() {
  obj.a.b.c = 888  // ✅ 触发，不管嵌套多深
}
```

### 3. oldValue 的问题（和 014 一样）

```ts
watch(person, (newValue, oldValue) => {
  // 修改 person.name 后：
  // newValue.name = '李四'
  // oldValue.name = '李四'  ← 一样！因为是同一个 Proxy 对象
})
```

> reactive 的 watch 无法正确获取 oldValue，因为新旧值指向同一个 Proxy 代理对象。

### 4. 整体替换用 Object.assign

```ts
function changePerson() {
  Object.assign(person, { name: '李四', age: 80 })
  // 不能写 person = { name: '李四', age: 80 }（010 讲过的原因）
}
```

## 三种情况对比总结

| | 情况一（013） | 情况二（014） | 情况三（015） |
|--|-------------|-------------|-------------|
| 数据类型 | ref 基本类型 | ref 对象类型 | reactive 对象类型 |
| 需要 deep | 不需要 | **需要** | 不需要（默认深度） |
| oldValue 正确 | ✅ | ❌ | ❌ |
| 监视地址变化 | ✅ | ✅ | 不适用 |
