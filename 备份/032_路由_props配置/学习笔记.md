# 032 路由：props 配置

## 本节目标

用路由的 `props` 配置，让路由参数自动作为 props 传给组件——组件不再需要 `useRoute()`。

## 相对 031 的变化

Detail.vue 的接收方式变了：

```ts
// 031：用 useRoute 接收
import { useRoute } from 'vue-router'
const route = useRoute()
// 模板：{{ route.params.id }}

// 032：用 defineProps 接收（像普通组件 props 一样）
defineProps(['id', 'title', 'content'])
// 模板：{{ id }}
```

路由配置中加了 `props` 配置项。

## 核心知识点

### props 的三种写法

**写法一：`props: true`**
```ts
{
  path: 'detail/:id/:title/:content?',
  component: Detail,
  props: true  // 把所有 params 参数作为 props 传给 Detail
}
```
→ Detail 收到 `id`、`title`、`content` 三个 props

**写法二：`props: 函数`（本节使用）**
```ts
{
  path: 'detail',
  component: Detail,
  props(route) {
    return route.query  // 把 query 参数作为 props 传给 Detail
  }
}
```
→ 函数收到 `route` 参数，返回什么对象，组件就收到什么 props

**写法三：`props: 对象`**
```ts
{
  path: 'detail',
  component: Detail,
  props: { a: 100, b: 200, c: 300 }  // 固定值
}
```
→ Detail 收到 `a`、`b`、`c` 三个固定 props

### 三种写法对比

| 写法 | 传什么 | 适用场景 |
|------|--------|----------|
| `true` | 自动传 params 参数 | 用 params 时 |
| 函数 | 自定义（可用 query/params/任意值） | 最灵活，本节用 |
| 对象 | 固定值 | 传静态数据 |

### Detail.vue 的变化

```ts
// 032 的 Detail.vue
defineProps(['id', 'title', 'content'])
// 不需要 import useRoute！
```

```html
<li>编号：{{ id }}</li>     <!-- 直接用，不需要 route.params.id -->
<li>标题：{{ title }}</li>
<li>内容：{{ content }}</li>
```

### 优势

- 组件**不依赖 vue-router**（不用 useRoute）
- 组件更**通用**（可以被路由用，也可以被其他组件用 props 调用）
- 代码更**简洁**
