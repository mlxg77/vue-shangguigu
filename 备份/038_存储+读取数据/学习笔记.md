# 038 存储 + 读取数据

## 本节目标

创建 Pinia store，把组件数据搬到 store 的 state 中，组件从 store 读取数据。

## 相对 037 的变化

新增 `store/` 目录，创建两个仓库文件。组件从"自己养数据"变成"从 store 借数据"。

## 核心知识点

### 1. defineStore 创建仓库

**store/count.ts**
```ts
import { defineStore } from 'pinia'

export const useCountStore = defineStore('count', {
  state() {
    return {
      sum: 6,
      school: 'atguigu',
      address: '宏福科技园'
    }
  }
})
```

**store/loveTalk.ts**
```ts
export const useTalkStore = defineStore('talk', {
  state() {
    return {
      talkList: [
        { id: '01', title: '今天你有点怪，哪里怪？怪好看的！' },
        { id: '02', title: '草莓、蓝莓、蔓越莓，今天想我了没？' },
        { id: '03', title: '心里给你留了一块地，我的死心塌地' }
      ]
    }
  }
})
```

### 2. defineStore 的参数

| 参数 | 作用 | 示例 |
|------|------|------|
| 第一个参数 | 仓库唯一 ID | `'count'`、`'talk'` |
| 第二个参数 | 配置对象 | `{ state, getters, actions }` |

### 3. state — 存数据的地方

```ts
state() {
  return { sum: 6 }  // 必须是函数，返回初始状态对象
}
```

> state 必须是**函数**（类似 data），防止多个实例共享同一份引用。

### 4. 组件中读取 store 数据

```ts
// Count.vue
import { useCountStore } from '@/store/count'
const countStore = useCountStore()

// 模板中直接用
<h2>当前求和为：{{ countStore.sum }}</h2>
```

```ts
// LoveTalk.vue
import { useTalkStore } from '@/store/loveTalk'
const talkStore = useTalkStore()

// 模板中
<li v-for="talk in talkStore.talkList" :key="talk.id">{{ talk.title }}</li>
```

### 5. 038 只读不写

加/减按钮和获取情话按钮的**方法体是空的**——这节课只讲"读"，"写"留到 039。

## 036 vs 038 数据来源对比

| 数据 | 036（组件内） | 038（store 中） |
|------|-------------|----------------|
| sum | `ref(1)` in Count.vue | `state.sum: 6` in count.ts |
| talkList | `reactive([...])` in LoveTalk.vue | `state.talkList: [...]` in loveTalk.ts |
