# 006 setup 语法糖

## 本节目标

从 `export default { setup() {} }` 切换到 `<script setup>` 语法糖，大幅简化代码。

## 相对 005 的变化

Person.vue 的 `<script>` 标签变了：

```ts
// 003-005 的写法（旧）
<script lang="ts">
  export default {
    name: 'Person',
    setup() {
      let name = '张三'
      function changeName() { name = 'zhang-san' }
      return { name, changeName } // 必须手动 return
    }
  }
</script>

// 006 的写法（新）
<script lang="ts" setup name="Person">
  let name = '张三'
  function changeName() { name = 'zhang-san' }
  // 不需要 return！不需要 export default！
</script>
```

## 核心知识点

### 1. `<script setup>` 做了什么

它是 Vue 编译器的语法糖，等价于：
- 自动把 `export default { setup() { ... } }` 包好
- 自动把顶层变量和函数 return 给模板
- **你只写逻辑，编译器帮你包装**

### 2. `name="Person"` 的作用

```html
<script lang="ts" setup name="Person">
```

替代了原来的 `name: 'Person'`。需要 `vite-plugin-vue-setup-extend` 插件支持（[package.json](../../package.json) 里有装）。

### 3. 语法糖省掉了什么

| 旧写法 | 语法糖 |
|--------|--------|
| `export default { ... }` | 不用写 |
| `setup() { ... }` | 不用写 |
| `return { name, age, ... }` | 不用写 |
| `components: { Person }`（子组件注册） | import 后直接用 |
| `name: 'Person'` | `name="Person"` |

### 4. 数据仍然不是响应式的

```ts
let name = '张三'  // 普通变量
function changeName() {
  name = 'zhang-san' // 值改了，页面不变
}
```

语法糖只是简化了写法，**没有解决响应式问题**。这要等 007 的 `ref`。

## 与后续课程的关系

从 006 开始，**所有组件都用 `<script setup>` 语法糖**，不再写 `export default`。007 解决响应式问题。
