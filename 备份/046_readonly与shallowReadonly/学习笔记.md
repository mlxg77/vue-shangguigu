# 046 readonly 与 shallowReadonly

## 本节目标

掌握 `readonly` 和 `shallowReadonly`——将响应式数据变为**只读**的 API。

## 核心知识点

### 1. readonly

`readonly` 创建一个**深层只读**的响应式对象——所有层级都不能修改。

```ts
let sum1 = ref(0)
let sum2 = readonly(sum1)     // sum2 是 sum1 的只读副本

function changeSum2() {
  sum2.value += 1             // ❌ 警告：target is readonly
}

function changeSum1() {
  sum1.value += 1             // ✅ 修改 sum1，sum2 也会同步更新
}
```

> `readonly` 是深层只读：如果包裹的是对象，内部所有属性也都不可修改。

### 2. shallowReadonly

`shallowReadonly` 创建一个**浅层只读**的响应式对象——只有第一层属性只读，深层属性仍可修改。

```ts
let car1 = reactive({
  brand: '奔驰',
  options: {
    color: '红色',
    price: 100
  }
})
let car2 = shallowReadonly(car1)

function changeBrand2() {
  car2.brand = '宝马'          // ❌ 第一层只读，不能修改
}

function changeColor2() {
  car2.options.color = '绿色'  // ⚠️ 能修改！第二层不是只读
}

function changePrice2() {
  car2.options.price += 10     // ⚠️ 能修改！第二层不是只读
}
```

### 3. 对比总结

| API | 只读层级 | 特点 |
|-----|---------|------|
| `readonly` | 深层（所有层级） | 任何修改都会被阻止 |
| `shallowReadonly` | 仅第一层 | 第一层只读，深层可改 |

### 4. 与 045 的呼应

| 045（浅层响应式） | 046（只读） |
|---|---|
| `shallowRef` | `readonly` |
| `shallowReactive` | `shallowReadonly` |

> 045 讲的是"浅层响应"（深层不响应），046 讲的是"只读保护"（不允许修改）。
> 两者都有"shallow"版本，"shallow" = 只作用于第一层。

### 5. 使用场景

- **readonly**：传递给子组件的数据不想被修改（如全局配置）
- **shallowReadonly**：第一层结构不可变，但允许修改深层细节
- 通常配合 `provide` 使用：`provide('data', readonly(data))`

| 按钮 | 操作 | 结果 |
|------|------|------|
| 点我sum1+1 | 修改 sum1（原始 ref） | ✅ sum1 和 sum2 都更新 |
| 点我sum2+1 | 修改 sum2（readonly） | ❌ 警告，不能修改 |

| 修改颜色(car2) | 修改 car2.options.color（shallowReadonly 第二层） | ⚠️ 可以修改 |
| 修改价格(car2) | 修改 car2.options.price（shallowReadonly 第二层） | ⚠️ 可以修改 |
