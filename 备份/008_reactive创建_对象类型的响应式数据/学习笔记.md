# 008 reactive 创建：对象类型的响应式数据

## 本节目标

用 `reactive` 把对象/数组变成响应式数据，适用于对象类型的场景。

## 相对 007 的变化

007 用 `ref` 处理基本类型；008 用 `reactive` 处理对象类型（对象、数组、嵌套对象）。

## 核心知识点

### 1. reactive 的基本用法

```ts
import { reactive } from 'vue'

// 对象
let car = reactive({ brand: '奔驰', price: 100 })
// 数组
let games = reactive([
  { id: '01', name: '王者荣耀' },
  { id: '02', name: '原神' },
  { id: '03', name: '三国志' }
])
// 嵌套对象
let obj = reactive({ a: { b: { c: 666 } } })
```

### 2. 不需要 `.value`（和 ref 的核心区别）

```ts
function changePrice() {
  car.price += 10         // ✅ 直接操作属性，不需要 .value
}
function changeFirstGame() {
  games[0].name = '流星蝴蝶剑'  // ✅
}
function changeObj() {
  obj.a.b.c = 999         // ✅ 深层属性也是响应式的
}
```

**为什么 reactive 不需要 `.value`？** 因为 `reactive()` 返回的是 Proxy 代理对象，直接代理了原对象的所有属性。而 `ref()` 返回的是包装对象，原始值藏在 `.value` 里。

### 3. 深层响应式

```ts
let obj = reactive({ a: { b: { c: 666 } } })
obj.a.b.c = 999  // 页面会更新，不管嵌套多深
```

`reactive` 会递归地把所有层级的属性都变成响应式。

### 4. reactive 只能用于对象类型

```ts
let name = reactive('张三')  // ❌ 不行！基本类型用 reactive 没有效果
let age = reactive(18)      // ❌
```

基本类型只能用 `ref`。

### 5. 模板中直接用属性名

```html
<h2>汽车信息：一辆{{ car.brand }}车，价值{{ car.price }}万</h2>
<li v-for="g in games" :key="g.id">{{ g.name }}</li>
```

## ref vs reactive 速查（007 vs 008）

| 对比项 | ref（007） | reactive（008） |
|--------|-----------|----------------|
| 适用类型 | 基本类型 + 对象 | **仅对象** |
| JS 中访问 | `xxx.value` | `xxx.属性` |
| 模板中 | 直接 `{{ xxx }}` | `{{ xxx.属性 }}` |
| 重新赋值 | `xxx.value = 新值` ✅ | `xxx = 新对象` ❌（010 会讲） |
