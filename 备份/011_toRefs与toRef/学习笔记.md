# 011 toRefs 与 toRef

## 本节目标

解决 reactive 对象**解构后丢失响应式**的问题。

## 问题背景

```ts
let person = reactive({ name: '张三', age: 18 })
let { name, age } = person  // ❌ 解构后 name、age 变成普通变量，不再是响应式
```

> reactive 对象解构时，会把值"拷"出来，断开与原对象的响应式联系。

## 核心知识点

### 1. toRefs —— 把整个对象的属性都变成 ref

```ts
import { reactive, toRefs } from 'vue'

let person = reactive({ name: '张三', age: 18 })
let { name, age } = toRefs(person)
// name 和 age 现在是 ref 对象，修改它们会同步到 person
```

- `toRefs(person)` 返回一个对象：`{ name: ref, age: ref }`
- 每个 ref 的 `.value` 指向 person 对应的属性
- 修改 `name.value` 等于修改 `person.name`，**双向同步**

```ts
function changeName() {
  name.value += '~'
  console.log(name.value, person.name)  // 两者同步
}
```

### 2. toRef —— 把单个属性变成 ref

```ts
import { toRef } from 'vue'

let nl = toRef(person, 'age')  // nl 是 person.age 的 ref 引用
console.log(nl.value)  // 18
```

- `toRef(对象, '属性名')` 只转一个属性
- 和 toRefs 的区别：toRefs 转全部，toRef 转一个

### 3. toRefs/toRef 的本质

```
person = reactive({ name: '张三', age: 18 })
           ↓
toRefs(person) → {
  name: ObjectRefImpl { value → person.name },  ← 引用，不是拷贝
  age:  ObjectRefImpl { value → person.age }
}
```

解构出来的 ref 不是新数据，而是**指向原对象属性的引用**，所以改一个两边都变。

### 4. 模板中的使用

```html
<h2>姓名：{{ person.name }}</h2>   <!-- 通过原对象访问 -->
<h2>年龄：{{ person.age }}，{{ nl }}</h2>  <!-- nl 是 toRef 的结果，模板自动解包 -->
```
