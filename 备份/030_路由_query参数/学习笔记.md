# 030 路由：query 参数

## 本节目标

在路由跳转时传递 query 参数，并在目标组件中接收。

## 相对 029 的变化

News.vue 的 RouterLink 从简单跳转改为**带 query 参数**，Detail.vue 开始**接收并展示参数**。

## 核心知识点

### 1. 传递 query 参数（两种写法）

**写法一：字符串拼接**
```html
<RouterLink :to="`/news/detail?id=${news.id}&title=${news.title}&content=${news.content}`">
  {{ news.title }}
</RouterLink>
```

**写法二：对象写法（推荐）**
```html
<RouterLink :to="{
  name: 'xiang',
  query: {
    id: news.id,
    title: news.title,
    content: news.content
  }
}">
  {{ news.title }}
</RouterLink>
```

对象写法更清晰，不需要手动拼字符串。

### 2. URL 中的表现

无论哪种写法，URL 都会变成：
```
/news/detail?id=asfdtrfay01&title=很好的抗癌食物&content=西蓝花
```

### 3. 接收 query 参数（Detail.vue）

```ts
import { useRoute } from 'vue-router'
import { toRefs } from 'vue'

let route = useRoute()       // 获取当前路由信息
let { query } = toRefs(route) // 解构出 query（保持响应式）
```

模板中使用：
```html
<li>编号：{{ query.id }}</li>
<li>标题：{{ query.title }}</li>
<li>内容：{{ query.content }}</li>
```

### 4. useRoute vs useRouter

| API | 作用 | 类比 |
|-----|------|------|
| `useRoute()` | 获取当前路由信息（path、query、params） | 读"我在哪" |
| `useRouter()` | 获取路由器实例（push、replace） | "我要去哪"（034 会讲） |

### 5. 为什么用 toRefs

```ts
let route = useRoute()
let { query } = toRefs(route)  // ← 保持响应式
```

直接 `let { query } = route` 会丢失响应式（011 讲过的解构问题）。用 `toRefs` 包裹后解构才能保持响应式。

> 也可以直接用 `route.query.id`，不解构。

## query 参数完整流程

```
News.vue 传参 → URL 变成 /news/detail?id=xxx&title=xxx → Detail.vue 用 useRoute().query 接收
```
