# 020 简单的回顾一下 TS

## 本节目标

回顾 TypeScript 的 `interface` 和 `type`，为后续 props 的类型约束做准备。

## 核心知识点

### 1. interface 定义接口

```ts
// types/index.ts
export interface PersonInter {
  id: string
  name: string
  age: number
}
```

`interface` 定义了一个对象的"形状约束"——凡是声明为 `PersonInter` 类型的对象，必须有 `id`（string）、`name`（string）、`age`（number）三个属性。

### 2. type 定义自定义类型

```ts
// types/index.ts
export type Persons = PersonInter[]
// 等价于：export type Persons = Array<PersonInter>
```

`type` 可以定义别名类型。`Persons` 表示一个数组，数组中每一项都必须符合 `PersonInter` 接口。

### 3. interface vs type

| | interface | type |
|--|-----------|------|
| 语法 | `interface X { ... }` | `type X = ...` |
| 描述对象形状 | ✅ 最常用 | ✅ 也可以 |
| 描述联合/交叉类型 | ❌ | ✅ `type A = string \| number` |
| 描述数组 | ❌ | ✅ `type Arr = X[]` |
| 扩展（继承） | `extends` | `&` 交叉类型 |

### 4. 在组件中使用类型

```ts
// Person.vue
import { type PersonInter, type Persons } from '../types'

// 单个对象，受 PersonInter 约束
let person: PersonInter = { id: '01', name: '张三', age: 60 }

// 数组，受 Persons 约束
let personList: Persons = [
  { id: '01', name: '张三', age: 60 },
  { id: '02', name: '李四', age: 18 },
  { id: '03', name: '王五', age: 5 }
]
```

### 5. import { type } 语法

```ts
import { type PersonInter, type Persons } from '../types'
```

`type` 关键字告诉编译器：只导入类型，不导入值。这在打包时会被完全擦除，不增加体积。

## 文件结构

```
020_简单的回顾一下TS/
├── types/
│   └── index.ts      ← 类型定义（interface + type）
├── components/
│   └── Person.vue     ← 使用类型
├── App.vue
└── main.ts
```

> 把类型定义抽到单独的 `types/` 目录是 Vue + TS 项目的常见做法，方便复用。
