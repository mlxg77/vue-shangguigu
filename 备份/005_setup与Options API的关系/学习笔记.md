# 005 setup 与 Options API 的关系

## 本节目标

验证 setup 和 Options API（data、methods）能否共存，以及共存时的优先级规则。

## 相对 004 的变化

Person.vue 中**同时存在** setup 函数和 Options API（data + methods）：

```ts
export default {
  name: 'Person',
  beforeCreate() { console.log('beforeCreate') },
  data() {
    return {
      a: 100,
      c: this.name,   // ← data 中可以访问 setup 返回的 name
      d: 900,
      age: 90         // ← 和 setup 中的 age 同名！
    }
  },
  methods: {
    b() { console.log('b') }
  },
  setup() {
    let name = '张三'
    let age = 18      // ← 和 data 中的 age 同名！
    // ...
    return { name, age, tel, changeName, changeAge, showTel }
  }
}
```

## 核心知识点

### 1. setup 和 Options API 可以共存

模板里同时使用了 setup 返回的 `{ name, age, changeName, ... }` 和 Options API 的 `{ a, b, c, d }`：

```html
<h2>姓名：{{name}}</h2>      <!-- 来自 setup -->
<h2>年龄：{{age}}</h2>        <!-- 来自 setup（同名时 setup 优先） -->
<button @click="changeName">修改名字</button>  <!-- 来自 setup -->
<hr>
<h2>测试1：{{a}}</h2>         <!-- 来自 data -->
<h2>测试2：{{c}}</h2>         <!-- 来自 data（值是 this.name = '张三'） -->
<h2>测试3：{{d}}</h2>         <!-- 来自 data -->
<button @click="b">测试</button>  <!-- 来自 methods -->
```

### 2. 执行顺序：setup 先于 data

```
setup() 执行 → beforeCreate → created → data() 执行 → ...
```

因为 setup 先执行，所以 data 中的 `c: this.name` 能拿到 setup 返回的 `'张三'`。

### 3. 同名属性：setup 优先

`age` 在 setup 中是 `18`，在 data 中是 `90`。页面显示的是 **18**（setup 赢）。

> 原因：setup 返回的对象会先被挂到组件实例上，data 的同名属性无法覆盖。

### 4. data 中可以通过 this 访问 setup 返回的数据

```ts
data() {
  return {
    c: this.name  // this.name 拿到的是 setup 返回的 '张三'
  }
}
```

但反过来，setup 中**不能**通过 this 访问 data（因为 setup 执行时 data 还没初始化）。

## 共存规则总结

| 问题 | 答案 |
|------|------|
| setup 和 data/methods 能共存吗？ | 能 |
| 谁先执行？ | setup → beforeCreate → data |
| 同名属性谁赢？ | setup 赢 |
| data 能访问 setup 的数据吗？ | 能（通过 this） |
| setup 能访问 data 的数据吗？ | 不能（this 是 undefined） |

## 与后续课程的关系

005 证明了 setup 和 Options API 可以混用（方便迁移老项目）。但从 006 开始，课程会切换到 `<script setup>` 语法糖，**不再写 `export default` 和 data/methods**，全面拥抱 Composition API。
