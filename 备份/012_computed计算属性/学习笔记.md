# 012 computed 计算属性

## 本节目标

掌握 `computed` 的两种写法（只读 / 可读可写），理解计算属性的缓存机制。

## 核心知识点

### 1. 只读计算属性（简写形式）

```ts
import { ref, computed } from 'vue'

let firstName = ref('zhang')
let lastName = ref('san')

let fullName = computed(() => {
  return firstName.value.slice(0,1).toUpperCase() + firstName.value.slice(1) + '-' + lastName.value
})
```

- `computed(() => {...})` 传入一个 getter 函数
- 返回一个只读的 ref，通过 `fullName.value` 访问
- 依赖（firstName、lastName）变化时自动重新计算

### 2. 可读可写计算属性（完整形式）

```ts
let fullName = computed({
  get() {
    return firstName.value.slice(0,1).toUpperCase() + firstName.value.slice(1) + '-' + lastName.value
  },
  set(val) {
    const [str1, str2] = val.split('-')
    firstName.value = str1
    lastName.value = str2
  }
})
```

- `get()`：读取 `fullName.value` 时调用
- `set(val)`：给 `fullName.value` 赋值时调用，val 是新值

```ts
function changeFullName() {
  fullName.value = 'li-si'  // 触发 set('li-si') → firstName='li', lastName='si'
}
```

### 3. 缓存机制

模板里写了 6 次 `{{ fullName }}`，但 computed 的 get **只执行一次**：

```html
全名：<span>{{fullName}}</span>  <!-- 第1次：触发 get -->
全名：<span>{{fullName}}</span>  <!-- 第2次：用缓存 -->
全名：<span>{{fullName}}</span>  <!-- 第3次：用缓存 -->
<!-- ...共6次，get 只调用1次 -->
```

> 只要依赖没变，computed 就返回缓存结果，不会重复计算。这是它和 `methods` 的核心区别。

### 4. 计算属性的响应式链路

```
firstName 变了 → fullName 的 get 重新执行 → 模板更新
```

## 两种写法对比

| | 简写形式 | 完整形式 |
|--|---------|---------|
| 语法 | `computed(() => {...})` | `computed({ get, set })` |
| 读写 | 只读 | 可读可写 |
| 适用场景 | 只需展示，不需修改 | 需要双向绑定/手动设置 |

## 与后续课程的关系

012 讲了 computed。013-017 会讲 `watch`——另一个响应式核心 API。区别：
- **computed**：根据依赖**算出新值**（返回值导向）
- **watch**：监听数据变化**执行副作用**（行为导向，不返回值）
