# 028 路由：命名路由

## 本节目标

给路由规则起名字，通过 `name` 跳转，而不是写死 `path`。

## 相对 027 的变化

路由配置中每条规则多了 `name` 属性：

```ts
// 027
{ path: '/home', component: Home }

// 028
{ name: 'zhuye', path: '/home', component: Home }
```

RouterLink 改用 `name` 跳转：

```html
<!-- 027：用 path -->
<RouterLink :to="{ path: '/about' }">关于</RouterLink>

<!-- 028：用 name -->
<RouterLink :to="{ name: 'xinwen' }">新闻</RouterLink>
```

## 核心知识点

### 1. 给路由命名

```ts
routes: [
  { name: 'zhuye',  path: '/home',  component: Home },
  { name: 'xinwen',  path: '/news',  component: News },
  { name: 'guanyu',  path: '/about', component: About },
]
```

`name` 必须全局唯一。

### 2. 用 name 跳转

```html
<RouterLink :to="{ name: 'xinwen' }">新闻</RouterLink>
```

等价于 `:to="{ path: '/news' }"` 或 `to="/news"`。

### 3. name vs path

| | name | path |
|--|------|------|
| 写法 | `:to="{ name: 'xinwen' }"` | `:to="{ path: '/news' }"` 或 `to="/news"` |
| 耦合度 | 低（路径改了不影响） | 高（路径改了要改所有引用） |
| 可读性 | 好（语义化名称） | 一般 |
| 推荐度 | ✅ 推荐 | 也可以 |

### 4. 命名路由的优势

```ts
// 如果以后把 /news 改成 /news-list
{ name: 'xinwen', path: '/news-list', component: News }
```

用 `name` 跳转的代码**完全不用改**，只需改路由配置中的 path。用 `path` 跳转则需要全局搜索替换。
