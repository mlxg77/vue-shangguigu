# 007 ref 创建：基本类型的响应式数据

## 本节目标

用 `ref` 把基本类型变量变成响应式数据，解决 006 中"改了值页面不变"的问题。

## 相对 006 的变化

```ts
// 006（非响应式）
let name = '张三'
let age = 18

// 007（响应式）
import { ref } from 'vue'
let name = ref('张三')
let age = ref(18)
```

## 核心知识点

### 1. ref 的基本用法

```ts
import { ref } from 'vue'

let name = ref('张三')  // 返回一个 RefImpl 对象
let age = ref(18)
```

`ref()` 把传入的值包装成一个 **RefImpl（引用实现）对象**，结构大致是：
```
RefImpl {
  value: '张三',   ← 真正的数据藏在 .value 里
  __v_isRef: true
}
```

### 2. JS 中操作 ref 必须 `.value`

```ts
function changeName() {
  name.value = 'zhang-san'  // ✅ 通过 .value 修改
}
function changeAge() {
  age.value += 1            // ✅
}
```

**为什么需要 `.value`？** 因为 `ref()` 返回的是对象，原始值放在 `.value` 属性上。Vue 通过拦截 `.value` 的 get/set 来实现响应式。

### 3. 模板中不需要 `.value`（自动解包）

```html
<!-- 模板里直接用变量名，Vue 自动解包 .value -->
<h2>姓名：{{ name }}</h2>   <!-- 显示"张三"，不是 RefImpl 对象 -->
<h2>年龄：{{ age }}</h2>
```

### 4. 对比：响应式 vs 非响应式

```ts
let name = ref('张三')    // 响应式 → 改了页面会变
let tel = '13888888888'  // 非响应式 → 改了页面不变
```

> tel 和 address 没有用 ref 包裹，仍然是普通变量。这形成了一个鲜明对比。

## 页面行为

- 点"修改名字"：**页面从"张三"变成"zhang-san"**（006 时不会变，007 修复了）
- 点"修改年龄"：**页面年龄 +1**
