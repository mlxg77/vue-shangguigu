# 003 setup 概述

## 本节目标

首次接触 Vue 3 的 `setup` 函数——Composition API 的入口，理解它和 Options API 的区别。

## 相对 002 的变化

| 变化点 | 002 | 003 |
|--------|-----|-----|
| 数据写在哪 | `data()` 里 | `setup()` 里（普通变量） |
| 方法写在哪 | `methods` 里 | `setup()` 里（普通函数） |
| 响应式 | data 自动响应式 | **setup 里的变量不是响应式的！** |
| this | 指向组件实例 | **undefined（Vue 3 在弱化 this）** |

## 核心知识点

### 1. setup 函数的位置和执行时机

```ts
export default {
  name: 'Person',
  beforeCreate() {
    console.log('beforeCreate')
  },
  setup() {
    console.log(this) // undefined
    // ...
    return { name, age, tel, changeName, changeAge, showTel }
  }
}
```

**执行顺序：** `setup()` → `beforeCreate()` → `created()` → ...

> setup 是整个组件中**最先执行**的，甚至比 beforeCreate 还早。

### 2. setup 中的数据不是响应式的

```ts
setup() {
  let name = '张三'   // 普通变量，不是响应式
  let age = 18
  let tel = '13888888888'

  function changeName() {
    name = 'zhang-san' // 值确实改了
    console.log(name)  // 控制台打印 'zhang-san'
    // 但页面不会更新！因为 name 不是响应式数据
  }
}
```

**关键结论：** setup 里的 `let` 变量只是普通 JavaScript 变量，改了值页面不会变。要让它响应式，需要用 `ref` 或 `reactive`（007、008 会讲）。

### 3. setup 必须返回数据，模板才能用

```ts
setup() {
  let name = '张三'
  function changeName() { name = 'zhang-san' }
  return { name, changeName } // 交出去 → 模板里才能 {{ name }} 和 @click="changeName"
}
```

不 return 的变量/方法，模板中完全访问不到。

### 4. setup 中的 this 是 undefined

Vue 3 刻意把 setup 中的 this 设为 undefined，**不再指向组件实例**。原因是 Composition API 鼓励你直接用闭包变量，而不是通过 this 访问。

## 页面行为

- 页面能正常显示 "张三"、"18"
- 点"修改名字"按钮：控制台打印 `zhang-san`，但**页面仍显示"张三"**（非响应式）
- 点"修改年龄"按钮：控制台打印 19，但**页面仍显示 18**
- 点"查看联系方式"按钮：正常弹窗（因为不涉及更新视图）
