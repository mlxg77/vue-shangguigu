# 031 路由：params 参数

## 本节目标

掌握 params 参数——直接写在 URL 路径中的参数（不同于 query 的 `?key=value` 形式）。

## 相对 030 的变化

路由路径中加了**占位符**，参数从 query 改为 params：

```ts
// 030：query 参数，路径是 /news/detail
{ path: 'detail', component: Detail }

// 031：params 参数，路径是 /news/detail/:id/:title/:content?
{ path: 'detail/:id/:title/:content?', component: Detail }
```

## 核心知识点

### 1. 路由路径中声明占位符

```ts
{
  name: 'xiang',
  path: 'detail/:id/:title/:content?',  // :id :title 是必填，:content? 是可选
  component: Detail
}
```

- `:id` — 必填占位符
- `:content?` — 可选占位符（加 `?`）

### 2. 传递 params 参数

**写法一：字符串拼接**
```html
<RouterLink :to="`/news/detail/${news.id}/${news.title}/${news.content}`">
```

**写法二：对象写法（推荐）**
```html
<RouterLink :to="{
  name: 'xiang',
  params: {
    id: news.id,
    title: news.title,
    content: news.content
  }
}">
```

> **注意：对象写法只能用 `name`，不能用 `path`！** 因为 params 需要根据命名路由的占位符来匹配。

### 3. URL 中的表现

```
/news/detail/asfdtrfay01/很好的抗癌食物/西蓝花
```

参数直接嵌在路径里，不像 query 那样在 `?` 后面。

### 4. 接收 params 参数

```ts
import { useRoute } from 'vue-router'
const route = useRoute()

// 模板中
<li>编号：{{ route.params.id }}</li>
<li>标题：{{ route.params.title }}</li>
```

## query vs params 对比

| | query（030） | params（031） |
|--|-------------|--------------|
| URL 格式 | `/detail?id=xxx&title=xxx` | `/detail/xxx/xxx` |
| 路由配置 | 不需要占位符 | 需要 `:id/:title` |
| 对象写法 | 可用 path 或 name | **只能用 name** |
| 参数可选 | 随意 | 需要 `?` 标记 |
| 美观度 | 一般 | 更简洁 |

## 与后续课程的关系

031 用 `useRoute().params` 接收参数。032 会用 props 配置让接收更优雅——不需要 useRoute。
