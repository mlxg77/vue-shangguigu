# 047 toRaw 与 markRaw

## 本节目标

掌握 `toRaw` 和 `markRaw`——从响应式系统中"逃逸"的 API。

## 核心知识点

### 1. toRaw——获取原始对象

`toRaw` 获取一个响应式对象的**原始（非代理）对象**。

```ts
import { reactive, toRaw } from 'vue'

let person = reactive({
  name: 'tony',
  age: 18
})

let rawPerson = toRaw(person)   // rawPerson 是原始对象，不是代理

// 修改 rawPerson 不会触发视图更新
rawPerson.age = 100             // 数据变了，但页面不更新
```

> `reactive` 返回的是 Proxy 代理对象，`toRaw` 把它还原为原始对象。
> 修改原始对象不会触发响应式更新。

### 2. markRaw——标记为永不响应

`markRaw` 标记一个对象，使其**永远不会**变成响应式。

```ts
import { reactive, markRaw } from 'vue'

let car = markRaw({ brand: '奔驰', price: 100 })
let car2 = reactive(car)        // reactive 不会代理 car，因为已被 markRaw 标记

console.log(car)                // 原始对象
console.log(car2)               // 原始对象（没有被代理）

car2.price += 10                // 数据变了，但页面不更新
```

### 3. toRaw vs markRaw

| | toRaw | markRaw |
|--|------|---------|
| 作用 | 从响应式对象中取出原始对象 | 标记对象永不被代理 |
| 方向 | 响应式 → 原始 | 原始 → 永不响应 |
| 影响原对象 | 不影响（原响应式对象仍然响应） | 永久标记 |
| 典型场景 | 临时需要原始对象（如深拷贝、性能优化） | 第三方库实例、不需要响应的大对象 |

### 4. 实际应用场景

```ts
// 场景一：第三方库实例不需要被代理
import mockjs from 'mockjs'
let mockJs = markRaw(mockjs)    // mockjs 是复杂对象，代理它没意义还有性能开销

// 场景二：需要对响应式数据做深拷贝（避免代理干扰序列化）
let person = reactive({ name: 'tony', age: 18 })
let rawPerson = toRaw(person)
let copy = JSON.parse(JSON.stringify(rawPerson))  // 深拷贝原始对象
```

### 5. 与 045/046 的关系

| 章节 | API | 作用 |
|------|-----|------|
| 045 | `shallowRef` / `shallowReactive` | 浅层响应（减少代理深度） |
| 046 | `readonly` / `shallowReadonly` | 只读保护（禁止修改） |
| 047 | `toRaw` / `markRaw` | 脱离响应式（取消代理） |

> 三者都是在特定场景下"弱化"响应式系统：
> - 045：不深层代理
> - 046：不让修改
> - 047：不代理 / 取出原始
